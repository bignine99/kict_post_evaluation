## 🔴 즉시 진행할 다음 작업 (1순위)

### 1. 별지 제4호(결과보고서) 구조화 추출 테스트
- **작업 내용**: 오늘 반영한 별지 4호의 실제 보고서 양식(PDF 템플릿과 동일한 구조적 형태) 프롬프트가 잘 동작하는지 전체 추출 파이프라인(🚀 전체 항목 생성)을 실행해 봅니다.
- **포인트 검증**: 
  - 서술형 항목들이 `가.`, `■`, `-`, `○` 등 보고서 기호를 사용해 깔끔하게 나오는지 확인
  - `[사진: OO전경]` 형태의 캡션(Placeholder)이 제 위치에 생성되는지 확인
  - 사람이 직접 내용을 편하게 고칠 수 있도록 ✏️ 편집/저장 기능이 문제없이 동작하는지 확인

### 2. 정답지(Ground Truth) PDF 비교 검증
- **작업 내용**: 실제 사람이 쓴 최종 사후평가서 파일(`.013_report_case/99. 1. 사후평가서(최종).pdf`)과 AI가 추출한 내용을 비교하여 47개 필드의 정확도를 끌어올립니다.
- **이슈 해결**: 로컬 환경에서 해당 정답지 PDF 문서를 파이썬 라이브러리(PyMuPDF)가 읽지 못해 무한 로딩(Hang)이 걸렸던 문제를 다른 라이브러리(pdfplumber, pypdf2 등)를 통해 우회하여 해결해야 합니다.

## 🟡 후속 개선 과제 (2순위)

### 3. 추출 결과 DB 영구 저장 (Persistence)
- **현재 상황**: 별지 3호와 4호에서 추출한 결과(데이터)를 새로고침(F5) 하면 프론트엔드의 화면 메모리에서 날아갑니다.
- **작업 내용**: 이미 백엔드에 만들어진 `evaluation_results` DB 테이블을 연동하여, AI가 힘들게 뽑아낸 결과를 영구적으로 저장하고 다음 날 접속해도 그대로 불러올 수 있도록 API와 프론트엔드 로직을 추가합니다.

### 4. 추출 정확도 개선 로직 강화
- 현재 "계약 공사비", "타당성조사비/실시설계비" 등 아주 세밀한 금액 분류를 할 때 다른 차수의 금액을 가져오는 현상 수정.
- `backend/app/services/parsing_engine.py`의 `query` 지시문을 좀 더 제한적이고 날카롭게 다듬어야 함.

### 5. PDF/클라이언트 내보내기 기능 (최종 형태 구현)
- 웹 화면에 잘 띄워진 이 보고서 내용을 최종적으로 사용자에게 다운로드 가능하게 하거나 PDF 파일로 깔끔하게 떨어지게 만드는 모듈(Puppeteer 등) 연동.
